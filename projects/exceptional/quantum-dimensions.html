Quantum dimensions

A pair \([r,s]\) denotes \(Q^r q^s-Q^{-r}q^{-s}\). Each compact formula is shown as a numerator list \(N(V)\) and denominator list \(D(V)\), with \(\dim_q(V)=N(V)/D(V)\).

Machine-readable data

JSON is a text format for labelled lists. A browser may display it either as a collapsible tree or under a Raw Data tab; these are two views of the same file.

quantum_dimensions.json contains 45 records under the heading quantum_dimensions. Of these, 44 have kind: pair_product. If numerator is the list [[ri,si]] and denominator is [[uj,vj]], the record means \[ \dim_q(V)=\mathtt{sign} \frac{\prod_i[r_i,s_i]}{\prod_j[u_j,v_j]}. \] The remaining record, for V5(0,1,0), has kind: polynomial_functor and stores the defining expression instead. The field label_involution gives the partner label.

quantum_dimensions_rational_level5.json contains the other 11 level-5 quantum dimensions. Each rational_function is the exact quotient in Q, q, stored as text because its expanded numerator has thousands of terms.

Python can read the live compact file without any additional package: 

import json
from urllib.request import urlopen

url = ("https://www.brucewestbury.uk/projects/exceptional/data/stored/"
       "quantum_dimensions.json")
with urlopen(url) as response:
    records = json.load(response)["quantum_dimensions"]

by_label = {record["label"]: record for record in records}
print(by_label["V2010"])

Replacing data/stored/quantum_dimensions.json in the URL by data/derived/quantum_dimensions_rational_level5.json reads the non-pair file in the same way.
